# Отчёт по лабораторной работе №5

## Тема
Изучение фреймворков для тестирования на примере GTest.

## Цели и задачи
- Создать CMakeLists.txt для библиотеки banking
- Создать модульные тесты на классы Transaction и Account
- Использовать mock-объекты
- Настроить сборочную процедуру на GitHub Actions

## Выполнение

### Структура проекта
- banking/ — библиотека с классами Account и Transaction
- tests/ — модульные тесты

### Класс Account
Реализованы методы: GetBalance, ChangeBalance, Lock, Unlock.
Методы Lock/Unlock защищают баланс от изменений без блокировки.

### Класс Transaction
Реализован метод Make для перевода средств между счетами.
Учитывается комиссия (fee), проверяются граничные условия.

### Тесты
Написано 11 тестов для классов Account и Transaction.
Использованы mock-объекты через GoogleMock.
Все 11 тестов прошли успешно (100% passed).

## Вывод
Изучен фреймворк GTest для модульного тестирования на C++.
Написаны тесты с использованием mock-объектов для класса Transaction.
Настроен CI через GitHub Actions.
